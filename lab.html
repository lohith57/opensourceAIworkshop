OCI Workshop Validation Guide

This document provides detailed validation steps after creating the VCN, Compute VM, and MySQL HeatWave DB System.

1. Validate Compute VM Access (SSH)

Reference: Oracle documentation – Connecting to a Linux Instance

Step 1: Get Public IP Address

  1. Login to OCI Console
  2. Navigate to Compute → Instances
  3. Select your VM instance
  4. Copy the Public access IP address

Step 2: Locate SSH Private Key

Unzip the file you used for resource creation(AI_Demo_infrastructure), Locate the .key file to authenticate to the VM.

Step 3: Set Key Permissions (macOS / Linux)

chmod 400 ~/Downloads/workshop-key.key

Step 4: Connect to VM

ssh -i ~/Downloads/workshop-key.key opc@<PUBLIC_IP>

Example:

ssh -i ~/Downloads/workshop-key.key opc@129.154.xx.xx

Step 5: Verify Connection

If successful, you should see:

[opc@workshop-vm ~]$
Default username for Oracle Linux is opc.

2. Validate HeatWave DB Connectivity from VM

Step 1: Get Private DB IP

  1. Go to Databases → MySQL HeatWave → DB Systems
  2. Select your DB System
  3. Copy the Private IP

If successful, database connectivity is working.

3. Connect to DB System via Visual Studio Code (Recommended Approach)

Since the DB is in a private subnet, connect through the VM using VS Code Remote SSH.

Step 1: Install Extensions

  • Search for MySQL Shell Extension and click on Install

Step 2: Connect VS Code to Heatwave

  1. Open VS Code
  2. Select the Mysql shell extension
  3. Click +New Connection
  4. Enter:
FieldValue
HostPrivate DB IP
Port3306
Usernameadmin
PasswordYour DB password

Select SSH tunnel

  1. enter username: opc@Public IP
  2. add the private key path

Step 3: Test Query from VS Code

SELECT NOW();

You should receive current DB server timestamp.

4. Final Validation Checklist

  • ✔ VM accessible via SSH
  • ✔ VS Code connected to HeatWave DB

Cline Installation in VS Code

In this section, users will configure access to OCI Generative AI and connect it with Cline inside Visual Studio Code.

Step 1: Create IAM Policy for Generative AI Access

Navigate to Identity & Security → Policies in the OCI Console.

Click Add Policy and create the following policy:

Allow any-user to use generative-ai-family in compartment poc-compartment

This policy provides users access to the OCI Generative AI service.

for details on IAM policy refer this doc .

Step 2: Create OCI Generative AI API Key

Navigate to:

Analytics & AI → Generative AI

Select:

  • Compartment: your workshop compartment
  • Region: Chicago

From the left-hand menu, select API Keys.

On the API Keys list page, click Create API Key.

  1. Enter a Name for the API key (required)
  2. (Optional) Enter a Description

Step 3: Install Cline in Visual Studio Code

Open Visual Studio Code.

Navigate to:

Extensions

Search for:

Cline

Click Install.

Step 4: Configure Cline Settings

Open the Cline settings.

Configure the following:

Field Value
Provider Open AI Compatible
Base URL https://inference.generativeai.us-chicago-1.oci.oraclecloud.com
Model ID ocid1.generativeaimodel.oc1.us-chicago-1.amaaaaaask7dceya3bsfz4ogiuv3yc7gcnlry7gi3zzx6tnikg6jltqszm2q
