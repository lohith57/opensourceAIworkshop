Lab 3: Prompt-Led Application Development

This lab introduces a reusable prompt template that helps users generate production-quality starter applications using AI. Each section of the template captures a specific application requirement.

Starter Application Generation Template

# STARTER APPLICATION GENERATION TEMPLATE

Create a production-quality {application_type} web application.

1. Business Objective
2. Frontend Requirements
3. Required Features
4. Backend Requirements
5. Database Requirements
6. AI / GenAI Features
7. Deployment Requirements
8. Important

Header Explanation (Brief)

1. Business Objective

Defines the business problem and the primary user goal that the application is solving.

2. Frontend Requirements

Specifies the UI framework, responsiveness, visual style, layout, and user experience expectations.

3. Required Features

Lists all functional modules such as login, dashboard, search, orders, feedback, and reports.

4. Backend Requirements

Defines the backend framework, project structure, logging, exception handling, and modular services.

5. Database Requirements

Ensures executable SQL scripts are generated including database creation, tables, keys, indexes, and seed data.

6. AI / GenAI Features

Defines vector search, summarization, chatbot, recommendations, and HeatWave GenAI use cases.

7. Deployment Requirements

Specifies where the app will be deployed and ensures startup instructions are generated.

8. Important

Emphasizes production-quality output, UI polish, deployment readiness, and reusable architecture.

Example Prompt for Users to Run

Role: Senior Full-Stack Engineer.
Task: Build a production-quality E-commerce Web Application for electronics using Python Flask and MySQL HeatWave.

1. Required Web Pages
Generate clean, modular HTML5/Bootstrap 5 templates for the following:

Home/Landing Page: Includes a sticky navigation bar (Home, Orders, Feedback, Logout), a premium hero banner, and a prominent search bar.

Product Grid: A responsive 3-column layout featuring elegant product cards. Each card must include a product image, title, price, star rating, a short description, and an "buy" button.

Login Page: A visually polished, centered authentication form.

Order History Page: A clean dashboard view listing previous purchases with status badges post login.

Product details page: Complete product details, along with feedback.

2. Database Requirements (MySQL HeatWave)
Provide a complete, executable SQL script. The database is private, so the schema must be optimized for a direct connection via db_utils.

products Table: Must include:

product_description (TEXT) for human-readable details.

description_vector (VECTOR) column to store embeddings for semantic search.

image_url (VARCHAR) to store paths for product images.

Standard fields: id, name, price, category, rating,quantity.

Other Tables: users, orders, and feedback with all necessary Primary and Foreign Keys.

Seed Data: Include a seed_data.sql section with 15+ realistic electronic products (laptops, phones, headphones) containing detailed descriptions and placeholder image URLs and 5+ examples for Orders and feedbacks.

3. AI Feature: Semantic Vector Search
Implement a native Vector Search capability using MySQL HeatWave:

SQL Implementation: Use the VECTOR_DISTANCE function to calculate similarity between the user's query and the description_vector column.

Results: The search must return the most relevant products based on meaning/context rather than just keyword matching.

4. Technical Architecture
Backend: Modular Python Flask structure using a db_utils.py file for database connection details with placeholders.

Frontend: Use Vanilla JavaScript for API calls. Ensure the UI is responsive, uses soft shadows, rounded corners, and follows a premium "Apple/Amazon" aesthetic.

Next Steps: Implementation & Deployment

Step 1: Sample SQL Statements for HeatWave GenAI Procedure

Provide the sample queries below to the assistant. This ensures the AI understands the exact schema and parameter requirements for calling your specific HeatWave GenAI routines, allowing it to generate accurate code that integrates seamlessly with your database procedures.

To generate Embeddings: 

SELECT sys.ML_EMBED_ROW("What is artificial intelligence?", JSON_OBJECT("model_id", "all_minilm_l12_v2")) into @text_embedding;

for semantic search

SELECT DISTANCE(STRING_TO_VECTOR("[1.01231, 2.0123123, 3.0123123, 4.01231231]"), STRING_TO_VECTOR("[1, 2, 3, 4]"), "COSINE");

Refer Mysql heatwave Doc

Pro Tip

  • Contextual Awareness: The AI doesn't know your custom procedure names (like ML_GENERATE) unless you define them.
  • Syntax Precision: HeatWave GenAI routines use specific JSON objects for tasks; providing a sample ensures the AI doesn't "hallucinate" the wrong parameters
  • Zero-Error Integration: It bridges the gap between the backend Python code and the SQL layer.

Step 2: Initialize Database in HeatWave SQL Editor

Log in to the HeatWave SQL Editor and run the generated Create Table and Seeded Data queries.

Reason: This step must be performed manually because the database is hosted on a private network. For security reasons, the AI code assistant cannot access your private database directly; it can only provide the scripts for you to execute within your secure environment.

Step 3: Modify Connection Details

Update the database host configuration in your application code:

  • Use localhost or 127.0.0.1 if you are performing local testing via an SSH tunnel.
  • Use the Private IP of the database when you are ready to deploy the application on a Virtual Machine within the same Cloud VCN.

Step 4: Environment Setup & Execution for local machine testing

Instruct your AI code assistant to do the following:

  1. Create a virtual environment (venv) and install required packages.
  2. Run the main Python file.
  3. Create the SSH tunnel command to connect your local environment to the private database: and share your private Key path

OR run the below command your self in a new terminal

ssh -i <private_key> -L 3306:<db_private_ip>:3306 opc@<vm_public_ip>

Important Note

It is not a strict syntax that must be followed word-for-word. Users are encouraged to build on this foundation, customize the headers, add domain-specific requirements, and expand the prompt based on their application use case.

The goal is to use this as a starting point for creating high-quality, production-ready applications through prompt-led development.
